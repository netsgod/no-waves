# no waves

Публичный репозиторий клиентской оболочки `no waves`.

Что находится здесь:
- Electron-клиент;
- GitHub Actions для публикации Windows-сборки в `GitHub Releases`;
- минимальные файлы для локальной сборки клиента.

Чего здесь нет:
- серверной части;
- Telegram-бота;
- базы данных;
- внутренних ключей, секретов и админских утилит.

Важно:
- приложение работает только с настроенным сервером `no waves`;
- доступ внутри приложения по-прежнему контролируется сервером и ключом;
- если сервер недоступен, клиент не сможет нормально запуститься;
- релизная сборка всегда ходит только на основной сервер и не читает `NOWAVES_SERVER_URL`.

## Запуск клиента

```bash
npm install
npm start
```

## Сборка Windows

```bash
npm run dist
```

## Обновления через GitHub

Клиент проверяет обновления автоматически и скачивает их из `GitHub Releases` этого же репозитория.

Чтобы выпустить новую версию:

1. Собери актуальный релизный клиент и выложи `no-waves-setup.exe`, `no-waves-setup.exe.blockmap` и `latest.yml` на `https://nowaves.ru/`.
2. Подними `version` в `package.json`.
3. Создай git tag вида `v1.0.7` и отправь его в GitHub.

GitHub Actions сам заберёт готовые артефакты с `nowaves.ru` и прикрепит их к `GitHub Releases`. После публикации пользователи увидят всплывающее окно, когда обновление будет найдено и скачано.

## Что реально защищает проект

Невозможно сделать Electron-клиент невзламываемым, если он лежит у пользователя на ПК. Защита держится на другом:

- серверная проверка ключа и сессии;
- отсутствие серверного кода и секретов в клиентском репозитории;
- хранение `client_id`, ключей и бизнес-логики только на сервере;
- возможность отозвать доступ на сервере без обновления клиента.
